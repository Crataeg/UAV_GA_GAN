# 20260327 AEU 数学公式加强版写作大纲包

## 目标

这套包不是在旧大纲上“多塞几条公式”，而是把当前工程重写成更接近 `AEU - International Journal of Electronics and Communications` 常见口味的数学论文结构：

- 先给出严格的系统模型与随机变量定义
- 再给出闭式或半闭式性能分析
- 然后把 `GA / GAN / measured-map` 放到优化与验证框架里
- 最后再回到当前工程真实结果，不做超出代码事实的夸大

## 为什么旧大纲不够

`20260326_最终论文写作包/05_论文写作入口/01_基于目前工程的行级写作大纲.md` 的优点是主线清楚，但它更像“工程解释稿”，主要问题有四个：

1. 公式大多停在定义层，没有形成可投稿的推导链。
2. `avg_total_deg` 和 `comm_deg` 被讲得太早，容易让审稿人误以为核心贡献只是评分函数。
3. `measured-map` 分支还缺少数学化表达，目前更像结果展示，不像验证模型。
4. `GA / GAN` 写法偏算法说明，缺少“先有明确优化问题，再给求解器”的论文语气。

## 这次重构后的核心主张

建议把全文重写为：

> 本文构建了一个面向城市低空无人机通信场景的 `SINR-first` 风险评估框架。该框架以统计城市形态、语义受限干扰部署、LoS/NLoS 路径损耗、FPC/CLPC 功率控制和 Nakagami 聚合干扰为基础，推导链路的 `outage / rate / BLER-adjusted throughput / energy efficiency`，并进一步在受限场景空间上进行最坏工况搜索与生成补样，同时利用公开实测通信场对算法发现高风险区域的能力进行验证。

## 已经可直接写进论文的工程结果

这些结果已经能直接支撑“算法有效性”部分：

### Synthetic 主线

- `GA` 的平均 `SINR` 为 `0.359 dB`
- `Random` 的平均 `SINR` 为 `3.645 dB`
- `GAN` 的平均 `SINR` 为 `5.634 dB`
- 因而当前工程事实是：
  - `GA` 最适合写成 worst-case 搜索器
  - `GAN` 更适合写成高风险分布学习与补样器

### Measured-map 主线

- `AERPAW` 上 `GA hit_rate_15m = 1.0`，`Random = 0.1167`
- `Dryad` 上 `GA hit_rate_15m = 1.0`，`Random = 0.1833`
- 因而当前工程事实是：
  - `GA` 在“发现实测差区域”上明显优于随机基线
  - `GAN` 在 measured-map 上有效，但仍弱于 `GA`

## 最重要的重写原则

- 主物理量必须是 `S`, `I`, `N`, `SINR`
- `outage / rate / throughput / EE` 是主结果层
- `comm_deg / avg_total_deg` 是优化层和工程汇总层
- `measured open data` 必须始终表述为 `measured communication field`，不能写成真实基站与干扰源全景真值图
