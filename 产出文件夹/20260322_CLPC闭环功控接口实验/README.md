# 闭环功控接口实验说明

## 1. 知识库回顾

- Obsidian 子知识库中没有独立的“闭环功控研究”专题节点。
- 当前可直接复用的本地材料主要有三类：
  - 代码中的 UE 上行 FPC/CLPC 接口。
  - `INTERFERENCE_SOURCE_TABLE.md` 中对 `cellular_ue_ul` 的公式说明。
  - 本地简报 `极简_工程仿真模型改进_CLPC_Nakagami.pptx`，其中提出了“时段-负载闭环功控”思路。
- 这次实验严格按工程现状执行：不新增完整时段负载 CLPC，只启用已经落地的 UE 上行闭环修正接口 `f(ΔTPC)`。

## 2. 实验设置

- 场景：4 架无人机，2 个地面站，8 个受控干扰源，关闭背景干扰源。
- 受控干扰源类型：全部固定为 `cellular_ue_ul`，确保触发 `_ue_ul_fpc_power_dbm()`。
- LoS 口径：`deterministic_if_map`。
- 聚合口径：`Nakagami`，并使用 `mean` 模式减少额外采样噪声。
- 对照组：
  - 开环基线：`f(ΔTPC)=0 dB`
  - 闭环上调：`f(ΔTPC)=+3 dB`
  - 闭环下调：`f(ΔTPC)=-3 dB`
- 说明：`±3 dB` 是接口启用演示值，用来验证闭环接口对结果的影响，不作为外场标定值。

## 3. 当前工程接口位置

- `UAV_GA.py:1986-2039`：UE 上行 FPC/CLPC 功率生成。
- `UAV_GA.py:2044-2091`：干扰源 profile 标定。
- `UAV_GA.py:2596-2749`：Nakagami 聚合干扰。
- `UAV_GA.py:2801-2889`：功率裕量、SINR、速率与中断输出。

## 4. 核心结果摘要

### 开环基线 0 dB
- 场景平均 UE 上行 TxPower：6.19 dBm
- 场景平均聚合干扰：-76.53 dBm
- 场景平均功率裕量：0.45 dB
- 场景平均 SINR：0.45 dB
- 场景平均综合退化度：0.3528
- 场景平均中断率：0.5000

### 闭环上调 +3 dB
- 场景平均 UE 上行 TxPower：9.11 dBm
- 场景平均聚合干扰：-73.78 dBm
- 场景平均功率裕量：-2.08 dB
- 场景平均 SINR：-2.08 dB
- 场景平均综合退化度：0.4029
- 场景平均中断率：0.5677

### 闭环下调 -3 dB
- 场景平均 UE 上行 TxPower：3.21 dBm
- 场景平均聚合干扰：-79.46 dBm
- 场景平均功率裕量：3.01 dB
- 场景平均 SINR：3.01 dB
- 场景平均综合退化度：0.2998
- 场景平均中断率：0.4219

## 5. 结论

- 当前工程中的闭环功控接口确实可以改变 UE 上行发射功率，并进一步改变聚合干扰、功率裕量、SINR 与综合退化度。
- 这说明“闭环功控”已经不是论文中的空词，而是可通过参数启用的工程接口。
- 但当前接口仍是静态增量形式，尚未演化到“按时段/负载/TPC 序列更新”的完整动态闭环系统。
