# 基于目前工程的行级写作大纲

这份大纲的目标，是把“论文每一节该写什么”直接对应到当前工程的代码位置、结果文件和写作依据。你可以把它当成一份按代码落地的论文骨架。

## 0. 题目与核心主张

建议总主张：

- 本工程建立了一个 `SINR-first` 的无人机通信性能劣化评估框架。
- 在功率域统一计算 `P_rx`、聚合干扰 `I`、噪声 `N`，得到 `SINR`，再推导 `outage / rate / BLER-adjusted throughput / EE`。
- `GA` 用于搜索最坏 synthetic 场景，`GAN` 用于学习高风险分布并生成相邻高风险场景。
- `AERPAW / Dryad` 作为 `measured communication field` 对照，用于证明算法的作用。

## 1. 引言

### 1.1 研究背景

建议写：

- 城市低空无人机通信受 `城市几何 + LoS/NLoS + 多类干扰源 + 能耗约束` 的共同影响。
- 许多工作强调链路预算或优化，但没有把 `power -> SINR -> 其余通信指标` 这条链统一写清。

可回指资产：

- `01_工程代码/UAV_GA.py:586`
- `02_行级技术解析/SINR_FIRST_MODEL_AND_LINE_MAP.md`
- `02_行级技术解析/RESEARCH_THOUGHT_TO_CODE_MAP.md`

### 1.2 问题定义

建议写：

- 我们研究的不是“随便定义一个打分公式”，而是：
  - 从功率域建模得到 `SINR`
  - 再从 `SINR` 推导出通信性能指标
  - 再用优化算法搜索风险场景

### 1.3 论文贡献

建议写成 4 点：

1. 建立 `power -> SINR -> outage/rate/throughput/EE` 的统一链路。
2. 将 `UE UL FPC + AP/BS CLPC + Nakagami aggregate interference` 纳入同一工程框架。
3. 比较 `GA / GAN / Random` 在 synthetic 场景中的作用。
4. 用 `AERPAW / Dryad measured field` 做非臆造对照。

## 3. SINR-first 通信性能评估模型

### 3.1 有用接收功率

代码锚点：

- `01_工程代码/UAV_GA.py:2564`

正文应写：

- `P_rx[dBm] = P_tx[dBm] - PL[dB] - extra_loss[dB] + 10log10(fading) + 10log10(activity)`

### 3.2 聚合干扰与噪声

代码锚点：

- `01_工程代码/UAV_GA.py:2657`
- `01_工程代码/UAV_GA.py:2846`

正文应写：

- `P_IN[mW] = I_y[mW] + N_y[mW]`
- `P_IN[dBm] = 10log10(P_IN[mW])`

### 3.3 功率裕量与 SINR

代码锚点：

- `01_工程代码/UAV_GA.py:2846`

正文应写：

- `M_db = P_rx[dBm] - P_IN[dBm]`
- `SINR = 10^(M_db/10)`

### 3.4 从 SINR 推导到其余指标

代码锚点：

- `01_工程代码/UAV_GA.py:2846`
- `01_工程代码/evaluate.py:101`
- `01_工程代码/kpi.py:56`
- `01_工程代码/kpi.py:99`

正文应写：

- `Outage = 1{SINR_dB < threshold}`
- `R = B log2(1 + SINR)`
- `Throughput = R * (1 - BLER(SINR))`
- `EE = R / P_total`

必须明确写：

- `4/5/6` 类源是 optional experimental source，不是默认背景事实

## 9. 结论

建议回到三点：

1. 本文建立了 `power -> SINR -> 其余通信指标` 的统一链路。
2. `GA` 在 synthetic worst-case search 和 measured poor-region discovery 中最稳健。
3. `GAN` 具有高风险分布学习价值，但当前更适合写成生成补样器。
